# README 客服工作台截图设计

## 目标

让招聘方在不安装 Docker、不运行本地服务的情况下，打开 GitHub 仓库首页即可直观看到本项目已有的 React 客服运营台。

## 已确认范围

- 使用当前项目实际运行的“客服工作台”桌面端界面，不制作概念图或独立展示站。
- 截图展示会话队列、聊天区、底部输入区和订单/商品侧栏，体现运营台的完整工作区布局。
- 图片保存为 `docs/images/support-console-overview.jpg`，并嵌入根目录 `README.md` 的“30 秒了解项目”之后。
- 图片中仅使用本地演示数据；不出现模型凭据、真实客户信息或外部平台密钥。

## 备选方案与取舍

1. **真实运行截图（采用）**：与现有代码一致，可信且无需额外维护展示站。
2. 手绘或生成式概念图：视觉可控，但无法证明仓库已有前端实现。
3. GitHub Pages 独立展示页：交互更强，但超出“有一张图片即可”的当前范围。

## 实施与验收

1. 启动现有 `support-console`，在桌面宽屏尺寸截取工作台。
2. 将 PNG 加入仓库，并使用相对路径在 README 中显示。
3. 本地核验 README 图片路径存在、图片可读取；不修改业务代码和运行行为。
4. 提交并推送到 `main`，使 GitHub 首页直接显示截图。

## 边界

- 该图片用于静态项目介绍，不替代本地 Docker 演示。
- 截图表现的仅是模拟电商渠道与演示数据；真实渠道接入和生产模型配置仍以 README 的“真实边界”为准。
