# 简历用 — AI 智能客服系统项目描述

---

## 简短版（一句话）

基于 Dify 自托管平台、FastAPI、Weaviate 与 n8n 搭建本地 AI 智能客服 MVP，具备知识检索、模拟订单/物流查询、人工兜底与审计闭环；目标接入抖店飞鸽客服系统。

---

## 详细版（简历项目经历用）

### AI 智能客服系统 | Dify / Docker / FastAPI / Weaviate / n8n

**项目背景：** 为电商店铺搭建 7×24 小时 AI 智能客服，自动回复常见商品咨询，查询订单/库存/物流信息，敏感场景自动转人工。

**核心职责与成果：**

**1. 技术选型与架构设计**
- 设计 Dify 可选接入与本地降级链路；公开仓库默认不配置模型供应商或 Dify App Key
- 设计包含 Dify、PostgreSQL、Redis、Weaviate、n8n、业务模拟服务、`agent-service` 与 React 运营台的 Docker Compose 本地闭环
- 使用 Docker Compose 实现一键部署

**2. 攻克网络限制下的部署难题**
- 曾处理受限网络下的镜像拉取与历史版本兼容问题；旧版 `0.15.3` 配置作为历史排障记录保留
- 当前 Compose 使用 Dify `1.14.2`；旧版 Apache OpenDAL 配置不参与当前运行

**3. 系统集成与功能实现**
- 实现本地策略、训练主题、RAG、订单模拟 Tool 与转人工闭环；Dify Chatflow 与模型提供商接入保留为目标环境配置
- 对接 Weaviate 向量数据库实现知识库 RAG 检索
- 搭建 db-simulator 模拟数据服务，验证订单/库存/物流查询链路
- 配置 n8n 工作流引擎，为抖店对接做好准备

**4. 抖店开放平台预研**
- 完成飞鸽客服系统 API 对接方案调研
- 识别出 API Key 申请需要营业执照资质

**技术栈：** Dify 1.14.2 容器 / Docker Compose / PostgreSQL / Redis / Weaviate / FastEmbed / n8n / Python / FastAPI / React
